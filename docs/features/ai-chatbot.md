# AIチャットボット
> ID: 25 | カテゴリ: function | プラン: premium

## 概要
画面右下にフローティングチャットバブルを常時表示し、クリックでチャットウィンドウを展開する機能。業種を問わず、顧客の定型質問（料金・営業時間・アクセス・サービス内容等）に自動応答し、営業時間外の機会損失を防ぐ。タイピングインジケーターで人間的な応答感を演出し、回答できない質問はお問い合わせフォームへエスカレーションする。セッション中のメッセージ履歴を保持し、初回訪問時には自動挨拶で訪問者をキャッチする。

## この機能の核
夜中でも自分の質問の答えが得られる

## 必須要件
- 画面右下固定のフローティングチャットバブル（position: fixed, z-index: 40以上）
- クリックで展開するチャットウィンドウ（幅340px / モバイルは画面幅100%）
- 定型FAQ応答: 業種ごとにカスタマイズ可能なキーワードマッチング
- 自由テキスト入力フィールド + 送信ボタン
- タイピングインジケーターアニメーション（ドット3つのパルス、800ms遅延）
- セッション中のメッセージ履歴保持（state管理）
- 最小化ボタン（バブルに戻す）
- 初回訪問時の自動挨拶メッセージ（3秒遅延で表示）
- 新着メッセージ時のバブルバッジ表示
- Framer Motionによるウィンドウ展開/折りたたみアニメーション
- クイックリプライボタン（FAQ候補をタップ可能なボタンで提示）

## リファレンスコード
```tsx
// warm-craft-pro の実装パターン（簡略版）
function AIChatbot() {
  const [isOpen, setIsOpen] = useState(false);
  const [messages, setMessages] = useState<{ role: "bot" | "user"; text: string }[]>([
    { role: "bot", text: "ようこそ！ご質問をお選びいただくか、お気軽にメッセージをどうぞ。" },
  ]);
  const [input, setInput] = useState("");

  const handleSend = () => {
    if (!input.trim()) return;
    const userMsg = input.trim();
    setInput("");
    setMessages((prev) => [...prev, { role: "user", text: userMsg }]);
    const match = FAQ_DATA.find((f) => userMsg.includes(f.q.slice(0, 4)));
    setTimeout(() => {
      setMessages((prev) => [
        ...prev,
        { role: "bot", text: match ? match.a : "詳しくはお問い合わせフォームからご連絡ください。" },
      ]);
    }, 800);
  };

  return (
    <>
      <motion.button
        onClick={() => setIsOpen(!isOpen)}
        className="fixed bottom-20 md:bottom-6 right-4 z-40 w-14 h-14 rounded-full bg-[accent] text-white shadow-lg"
        whileHover={{ scale: 1.05 }}
        whileTap={{ scale: 0.95 }}
      >
        {isOpen ? <X className="w-5 h-5" /> : <MessageCircle className="w-6 h-6" />}
      </motion.button>
      <AnimatePresence>
        {isOpen && (
          <motion.div
            className="fixed bottom-36 md:bottom-24 right-4 z-40 w-[340px] max-h-[480px]"
            initial={{ opacity: 0, y: 20, scale: 0.95 }}
            animate={{ opacity: 1, y: 0, scale: 1 }}
            exit={{ opacity: 0, y: 20, scale: 0.95 }}
          >
            {/* Header + Messages + Quick replies + Input */}
          </motion.div>
        )}
      </AnimatePresence>
    </>
  );
}
```

## 3層チェック

> この機能の核: **夜中でも自分の質問の答えが得られる**
> 核が実現されているかどうかを、3層で検証する。

### Layer 1: 機能チェック（動くか）— 30点

技術的に正しく動作するかの検証。全項目PASSが前提条件。1つでもFAILなら全体70点未満（作り直し）。

| # | チェック項目 | 合格基準 | FAIL例 |
|---|---|---|---|
| F-1 | バブル表示 | 画面右下にフローティングバブルが `fixed z-40` で常時表示 | バブルが表示されない。スクロールで消える |
| F-2 | ウィンドウ展開 | バブルクリックでチャットウィンドウがAnimatePresence付きで展開 | クリックしても開かない。アニメーションなし |
| F-3 | FAQ応答 | 定型FAQキーワードに対して正しい回答がマッチして返される | キーワードが一致しても回答が返らない。マッチングロジックが壊れている |
| F-4 | エスカレーション | マッチしない入力に「お問い合わせフォームから」の誘導が表示される | ループして同じ答え。または無応答 |
| F-5 | タイピングインジケーター | 応答前に800msのドット3つパルスアニメーションが表示される | いきなり回答表示。人間的な応答感なし |
| F-6 | メッセージ履歴 | セッション中のメッセージ履歴がstate管理で保持されている | ウィンドウ閉じ→開きで履歴消失 |
| F-7 | クイックリプライ | FAQ候補のタップ可能なボタンが表示され、タップで正しく応答する | クイックリプライなし。ボタンが反応しない |
| F-8 | 最小化 | 最小化ボタンでバブルに戻り、未読バッジが表示される | 閉じ方がわからない。未読バッジなし |
| F-9 | 自動挨拶 | 初回訪問時に3秒遅延で挨拶メッセージが表示される | 挨拶なし。即座に表示されて唐突 |
| F-10 | z-index | バブル/ウィンドウのz-40が他のfixed要素と競合しない | ヘッダーの下に隠れる。SP電話バーと重なる |

### Layer 2: UXチェック（迷わないか）— 40点

ユーザーが操作に迷わず、ストレスなく目的の情報に辿り着けるかの検証。

| # | チェック項目 | 合格基準 | 閾値 | 配点 |
|---|---|---|---|---|
| U-1 | 応答速度 | タイピングインジケーター含め**2秒以内に回答表示** | ユーザー送信→タイピング800ms→回答表示。合計2秒以内 | 10点 |
| U-2 | クイックリプライの発見しやすさ | チャットを開いた瞬間に**「何を聞けばいいか」がわかる** | 挨拶メッセージ直後にクイックリプライボタンが3〜5個並ぶ | 8点 |
| U-3 | 閉じ方の明確さ | チャットウィンドウの**閉じ方が直感的にわかる** | Xボタンまたはバブル再タップで閉じる。ESCキーも対応 | 8点 |
| U-4 | モバイル全画面体験 | スマホで**全画面チャットが快適に操作できる** | キーボード表示時にメッセージ欄が押し上げられない。入力欄が常に見える | 7点 |
| U-5 | FAQ→フォームのエスカレーション | 回答できない質問で**即座にお問い合わせ導線**が提示される | 「詳しくはこちらからお問い合わせください」+フォームリンク | 7点 |

### Layer 3: 価値チェック（答えが得られるか）— 30点

この機能の核「夜中でも自分の質問の答えが得られる」が実現されているかの検証。**ここが80点と90点を分ける。**

| # | チェック項目 | 合格基準 | 90点の例 | 80点の例 | FAIL例 |
|---|---|---|---|---|---|
| V-1 | 答えが実際に得られる | ユーザーの質問に対して**具体的な答え**が返る | 「料金は坪単価60〜80万円。30坪で1,800〜2,400万円です」 | 「料金はプランにより異なります。目安は○○円〜」 | 「お気軽にお問い合わせください」で具体情報ゼロ |
| V-2 | 的外れ回答時の誘導 | 回答できない場合に**具体的な次のアクション**を提示する | 「その質問にはお答えできません。お電話(xxx-xxxx)またはフォームからどうぞ」 | 「お問い合わせフォームからご連絡ください」 | 「わかりません」で終了。導線なし |
| V-3 | 「便利だ」と感じるか | チャットを使って**「電話しなくて済んで助かった」**と感じる | 営業時間外に料金・アクセス・予約方法が全て回答される | 基本的な質問には答えられる | 何を聞いても「お問い合わせください」で電話と変わらない |
| V-4 | 業種のFAQカバレッジ | その業種で**ユーザーが実際に聞く上位5質問**に全て回答できる | 建築:坪単価/工期/対応エリア/保証/見学会。全て具体的数値付き | 上位5質問のうち3つに回答。2つは「お問い合わせ」 | 5質問中1つしか答えられない |
| V-5 | 24時間対応の実感 | 深夜にアクセスしても**「対応してもらえている」感覚**がある | 営業時間外でも全FAQに回答+「翌営業日に担当者からご連絡します」 | FAQには答えるが営業時間外メッセージなし | 「営業時間内にお電話ください」で即終了 |
| V-6 | FAQ定数の汎用性 | FAQ_DATA配列の差し替えだけで**任意の業種に対応**できる | keywords/question/answer/categoryの構造で、業種定数を差し替えるだけ | 構造は汎用的だが一部ハードコード | 業種固有のロジックが関数内にベタ書き |

### 判定ルール

| スコア | 判定 | アクション |
|---|---|---|
| 90〜100 | **PASS** | そのまま次の機能へ。核が実現されている |
| 80〜89 | **CONDITIONAL** | Layer 2 or 3 の減点箇所を特定し修正。修正後に再チェック |
| 70〜79 | **FAIL** | Layer 1 は通るがUXか価値が不足。原因を記録し、該当層を作り直し |
| 0〜69 | **CRITICAL FAIL** | Layer 1 がFAIL。機能として動いていない。全体を作り直し |

### 採点の具体例

**90点の実装:**
- バブル+ウィンドウ+FAQ応答+タイピングインジケーター+クイックリプライが全て動作（L1: 30/30）
- 応答2秒以内。クイックリプライが即座に目に入る。モバイル全画面でキーボードと干渉しない（L2: 35/40）
- 「坪単価60〜80万円」等の具体的数値付き回答。上位5質問を全カバー。深夜でも「翌営業日にご連絡」（L3: 25/30）

**80点の実装:**
- チャット機能は全て動作する（L1: 30/30）
- クイックリプライあり。モバイル対応OK。応答速度2秒以内（L2: 28/40）
- FAQには答えるが具体的数値が少ない。5質問中3つカバー（L3: 22/30）

**70点の実装:**
- チャットは動く。クイックリプライなし（L1: 30/30）
- テキスト入力のみで何を聞けばいいか不明。モバイルでキーボードと干渉（L2: 22/40）
- ほとんどの質問に「お問い合わせください」。具体的回答なし（L3: 18/30）

### この機能固有の重要判定ポイント

1. **「お気軽に」カウント**: 回答に「お気軽にお問い合わせください」「ケースバイケース」系が含まれる回数をカウント。**FAQ全体の50%以上が「お問い合わせ」回答ならV-1自動FAIL**。チャットボットの意味がない
2. **クイックリプライの必須性**: テキスト入力だけでは「何を聞けばいいかわからない」。**クイックリプライなしはU-2が0点**
3. **z-index設計**: warm-craftのSP電話バー(z-40)+チャットバブル(z-40)が同時表示される場合の**位置オフセット設計が必須**。重なったら両方タップ不能
