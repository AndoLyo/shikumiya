## 1. プロジェクト概要

### 1.1 事業名
`[固有]` しくみや（by Lyo Vision）

### 1.2 事業概要
`[固有]` 建築業（工務店・建設会社・設計事務所）向けのホームページ制作SaaS。初期費用0円・月額3,000円からで、写真を送るだけでプロ品質のホームページが翌日完成する。AI自動化により1人運営でも高品質・低価格を実現。

### 1.3 ミッション
`[固有]` 「写真を送るだけ。あとは全部おまかせ」— パソコンが苦手な建築業の社長でも、写真を送るだけでホームページが持てる。制作・更新・運用のすべてを引き受け、本業に集中できる環境を提供する。

### 1.4 汎用フレームワーク
`[汎用]` このサービスは以下の構造で成立する：
```
専門スキル（サイト制作）
  × AI自動化（構築・更新・カスタマイズの自動化）
  × 特定ターゲット（建築業の中小零細企業）
  × 低価格（制作費0円 + 月額サブスク）
  = 既存市場の空白地帯を取る
```
別ターゲットに展開する場合、上記の4要素を入れ替えるだけで同じ構造が使える。

## 2. ターゲット定義

### 2.1 プライマリターゲット
`[固有]`
- **属性**: 建築業（工務店・建設会社・設計事務所）の経営者
- **状況**: ホームページがない、または10年前に作ったまま放置している
- **ペイン**: 「ホームページが必要なのはわかってる。でも高い・面倒・誰に頼めばいいかわからない」
- **ゲイン**: 初期費用0円で自社のホームページが持てる。更新も全部おまかせ

### 2.2 ターゲットセグメント
`[汎用]` ターゲットは以下の3層に分かれる：

| セグメント | 特徴 | アプローチ |
|---|---|---|
| **(a) HPなし層** | ホームページを持っていない。必要性は感じているが手つかず | 「制作費0円。写真送るだけ」で訴求 |
| **(b) HP放置層** | 10年前のHPがある。古いまま放置。更新方法もわからない | 「今のHP、見られてますか？」で問題提起 |
| **(c) HP高額不満層** | 制作会社に頼んだが高額。月額管理費も高い。乗り換えたい | 「制作費0円、月3,000円から」で価格訴求 |

### 2.3 ペルソナ（メインターゲット）
`[固有]`
- 55歳、工務店の社長（従業員5名以下）
- パソコンは苦手。スマホは使える。SNSはほぼ見ない
- Google検索で「工務店 ホームページ 安い」等で探す層
- 息子や事務員に「ホームページくらい作れよ」と言われている
- 制作会社に見積もりを取ったら50万円と言われて諦めた
- 「安くて、全部やってくれるなら頼みたい」

### 3.5 新規申込フロー（/start）

**概要**: セルフサービス申込。相談不要、最短3分で申込完了

**ステップ**:
1. **STEP 0**: 業種を選ぶ（工務店 / 建設会社 / 設計事務所 — ワンクリック）
2. **STEP 1**: テンプレートを選ぶ（iframeプレビュー付き3択）
3. **STEP 2**: 会社名を入力（リアルタイムでデモサイトに反映表示）
4. **STEP 3**: ドメイン決定 + メールアドレス入力 → 申込完了

**処理フロー**:
```
1. ユーザーが /start で申込情報を入力
2. 申込完了 → Stripe Webhook → サイト生成パイプライン起動
3. テンプレートベースでサイト自動生成
4. GitHub リポジトリ作成 → Vercelプロジェクト作成 → 自動デプロイ
5. 独自ドメイン設定
6. 完了メールをユーザーに送信（サイトURL付き）
7. 決済はサイト完成後に開始（Stripeサブスクリプション）
```

**出力（ユーザーが受け取るもの）**:
- 公開されたホームページ（独自ドメイン or 仮URL）
- 完了メール
- 顧客管理ページへのアクセス情報

**非機能要件**:
- 生成時間: 目標翌日（最大3営業日）
- 可用性: Vercelの稼働率に依存（99.99%）
- セキュリティ: ユーザーの画像・個人情報は最小限のみ保持

### 3.6 顧客管理ページ（/member/[orderId]/）

**概要**: 顧客が自分のサイトの状態確認・編集依頼を行うページ

**機能**:
- サイドバーナビゲーション + プラン別🔒表示
- ダッシュボード（サイト状態の確認）
- 編集依頼（選択式UI + 画像クロップ機能）
- 依頼履歴
- サイト機能一覧（閲覧専用）
- アカウント設定（プラン変更モーダル）
- アップグレード導線

**編集依頼の処理フロー**:
```
テキスト/画像変更 → GitHub API経由で即反映 → Vercel自動デプロイ
レイアウト/機能変更 → Claude API経由で自動処理 → スコアリング → 自動承認 or 手動確認
```

### 3.7 運営管理ページ（/admin/）

**概要**: Lyo（運営者）用の管理ダッシュボード

**機能**:
- ダッシュボード（MRR / 顧客数 / 未対応依頼数）
- 編集依頼キュー（完了 / 修正指示 → Claude API送信）
- 顧客一覧

## 4. 非機能要件

### 4.1 パフォーマンス
- トップページのLCP: 2.5秒以内
- テンプレートサイトのLCP: 2.0秒以内
- Core Web Vitals: すべてGreen

### 4.2 SEO / AI SEO
- 構造化データ: JSON-LD（Service, FAQ, LocalBusiness, Organization）
- セマンティックHTML: 正しい見出し階層、リスト、テーブル
- meta description: 各ページに固有
- OGP: 動的生成（opengraph-image.tsx）
- 「工務店 ホームページ制作」「建設会社 ウェブサイト 安い」「建築業 HP作成」等で上位表示を目指す

### 4.3 アクセシビリティ
- WCAG 2.1 AA準拠（最低限）
- キーボードナビゲーション対応
- スクリーンリーダー対応（alt, aria-label）
- フォーカス表示
- 文字サイズ: ターゲット層（55歳前後）を考慮し、本文16px以上

### 4.4 セキュリティ
- HTTPS（Vercel自動）
- Stripe決済（PCI DSS準拠）
- ユーザー画像: Vercel上に保存（第三者アクセス不可）
- 個人情報: メールアドレス・会社情報のみ保持

## 5. 制約事項

### 5.1 技術的制約
- Next.js + Tailwind CSS + Framer Motion
- Vercelでホスティング
- Stripe決済（サブスクリプション）
- Claude API（編集依頼の自動処理）
- GitHub API（サイトデータの更新・デプロイ）

### 5.2 ビジネス的制約
- 運営者: 1人（Lyo）
- AI構築の事実は対外的に伏せる
- 初期は赤字許容（顧客獲得優先）
- 無料相談は不要 — セルフサービス申込に統一

### 5.3 テンプレート構成
- **3業種**: 工務店(warm-craft) / 建設会社(trust-navy) / 設計事務所(clean-arch)
- **3プラン**: ライト / ミドル(-mid) / プレミアム(-pro)
- **合計9テンプレート**
- パス: `src/app/portfolio-templates/` + `src/components/portfolio-templates/`

### 5.4 法的要件
- 特定商取引法に基づく表示ページ（/legal）
- プライバシーポリシー（/privacy）
- 利用規約
