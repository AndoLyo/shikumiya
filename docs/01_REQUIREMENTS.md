## 1. プロジェクト概要

### 1.1 事業名
`[固有]` しくみや（by Lyo Vision）

### 1.2 事業概要
`[固有]` AIアーティスト向けのオリジナルギャラリーサイト制作サービス。フォーム入力だけでサイトが自動生成される。月額プランではAIによる継続的なカスタマイズを提供。

### 1.3 ミッション
`[固有]` 「作品に集中できる仕組みを作る」— 作品を作る以外のこと（サイト構築・運用）をすべて引き受け、クリエイターが創作に集中できる環境を提供する。

### 1.4 汎用フレームワーク
`[汎用]` このサービスは以下の構造で成立する：
```
専門スキル（サイト制作）
  × AI自動化（構築・カスタマイズの自動化）
  × 特定ターゲット（AIアーティスト）
  × 低価格（自動化によるコスト削減を価格に反映）
  = 既存市場の空白地帯を取る
```
別ターゲットに展開する場合、上記の4要素を入れ替えるだけで同じ構造が使える。

## 2. ターゲット定義

### 2.1 プライマリターゲット
`[固有]`
- **属性**: AI画像生成者（AIアーティスト）
- **状況**: XやpixivにAIアート作品を投稿しているが、自分のサイトは持っていない
- **ペイン**: 作品が流れていく。まとめて見せる場所がない。サイトを作るのはめんどくさい
- **ゲイン**: 自分だけのギャラリーサイトが持てる。作品が資産として蓄積される

### 2.2 ターゲットセグメント
`[汎用]` ターゲットは以下の3層に分かれる：

| セグメント | 特徴 | アプローチ |
|---|---|---|
| **(a) やりたくない層** | サイトの必要性は理解。自分では絶対やりたくない | 「全部やります」で訴求 |
| **(b) やりたいが方法不明層** | 興味はある。何から始めればいいかわからない | 「5分でできます」で訴求 |
| **(c) 潜在ニーズ層** | サイトの必要性に気づいていない | 「もったいなくない？」で問題提起 |

### 2.3 ペルソナ（最初の1人目）
`[固有]` ※実装前に具体化する。以下は仮設定：
- 30代、AI画像生成歴1年
- Stable Diffusion / Midjourney / NovelAI を使用
- Xに毎日1〜3枚投稿、フォロワー500〜3,000
- pixivにも投稿しているが、作品がバラバラ
- 「ポートフォリオ作りたいな」と思ったことはある。でもやっていない

### 3.2 S-01: サイト自動生成（MVP）

**概要**: フォーム入力→サイト自動生成→Vercelデプロイ

**入力（ユーザーが行うこと）**:
1. 名前 / 活動名
2. ジャンル選択（イラスト / 写真 / デザイン / AIアート / その他）
3. テンプレート選択（10種から1つ）
4. 作品画像アップロード（3〜10枚）
5. 自己紹介文（任意）
6. SNSリンク（任意）
7. メールアドレス（Stripe決済用）

**処理フロー**:
```
1. Stripeで¥980決済
2. 決済完了 → Stripe Webhook → GAS受信
3. GASがフォームデータを整形
4. Claude Code CLIにシステムプロンプト+データを送信
5. Claude Codeがテンプレートベースでサイト生成
6. 画像を最適化してpublic/に配置
7. Vercelにデプロイ（git push → 自動ビルド）
8. 完了メールをユーザーに送信（サイトURL付き）
```

**出力（ユーザーが受け取るもの）**:
- 公開されたサイト（https://username.vercel.app）
- 完了メール

**非機能要件**:
- 生成時間: 目標5分以内（最大30分）
- 可用性: Vercelの稼働率に依存（99.99%）
- セキュリティ: ユーザーの画像・個人情報は最小限のみ保持

### 3.3 S-02: AIカスタマイズ

**概要**: 編集フォーム入力→AI確認→承認→AI自動実装

**入力**:
1. 編集したい内容（テキスト入力）
2. 参考画像（任意）

**処理フロー**:
```
1. ユーザーが編集フォームに入力
2. AIが内容を解釈し「この変更でよろしいですか？」と返答
3. ユーザーが承認
4. Claude Code CLIがサイトを改変
5. Vercelに自動デプロイ
6. 完了通知
```

**制約**:
- 月の改変回数: 制限なし（自動のため）
- 大規模リニューアル: プレミアムプランに誘導

### 3.4 S-04: プレビュー体験

**概要**: LP上で画像1枚アップロード→テンプレートにはめ込み→Afterを3秒で表示

**目的**: 「めんどくさい」を超える。購入前にAfterを体験させる

**処理**:
- クライアントサイドで完結（サーバー不要）
- Canvas APIまたはCSS合成でテンプレートに画像をはめ込み
- 決済前の体験なので個人情報不要

## 4. 非機能要件

### 4.1 パフォーマンス
- LPのLCP: 2.5秒以内
- テンプレートサイトのLCP: 2.0秒以内
- Core Web Vitals: すべてGreen

### 4.2 SEO / AI SEO
- 構造化データ: JSON-LD（Service, FAQ, Person, Organization）
- セマンティックHTML: 正しい見出し階層、リスト、テーブル
- meta description: 各ページに固有
- OGP: 動的生成（opengraph-image.tsx）
- 「AIアーティスト ポートフォリオサイト」「AIアート ギャラリー」等で上位表示を目指す

### 4.3 アクセシビリティ
- WCAG 2.1 AA準拠（最低限）
- キーボードナビゲーション対応
- スクリーンリーダー対応（alt, aria-label）
- フォーカス表示

### 4.4 セキュリティ
- HTTPS（Vercel自動）
- Stripe決済（PCI DSS準拠）
- ユーザー画像: Vercel上に保存（第三者アクセス不可）
- 個人情報: メールアドレスのみ保持。GASスプレッドシートで管理
