## 1. ファネル全体像

`[汎用]` この構造は「SEO集客→LP→即申込→月額サブスク→アップセル」のファネル。ターゲットやサービスを変えても構造は同じ。

```
認知（TOFU）
  SEO（「建築業 ホームページ 安い」等のGoogle検索）
  建築向けLP（/lp/construction）への流入
    │
    ▼
興味（MOFU）
  /lp/construction LP訪問
  テンプレートデモページ（/portfolio-templates/*）
  Before/After:「写真を送る前 → 完成サイト」
  「うちの会社もこうなるのか…」← 感情が動く
    │
    ▼
申込（BOFU）
  /start ページ（相談なし・即申込）
  STEP 0: 業種選択（ワンクリック）
  STEP 1: テンプレート選択（iframeプレビュー付き3択）
  STEP 2: 会社名入力 → リアルタイムデモサイト表示
  STEP 3: ドメイン + メアド → 申込完了
  「自分の会社のサイトがもうできてる！」← 感動
    │
    ▼
アップセル
  ライト（¥3,000/月）→ ミドル（¥8,000/月）
  ミドル → プレミアム（¥15,000~/月）
  会員ダッシュボード（/member/[orderId]/settings）から
    │
    ▼
継続（Retention）
  月次更新（ライト1回/ミドル3回/プレミアム無制限）
  編集依頼フォーム
```

## 2. 各段階の設計

### 2.1 認知（TOFU）

`[固有]`

| チャネル | コンテンツ | 目的 |
|---|---|---|
| SEO（Google検索） | 「建築業 ホームページ 安い」「工務店 サイト 作成」等 | 検索流入の獲得 |
| 建築向けLP | /lp/construction | SEOの着地先・第一印象 |
| テンプレデモ | /portfolio-templates/* の各デモページ | 「こんなサイトが持てる」の実感 |

`[汎用]` 認知段階では「売る」のではなく「見せる」。完成サイトのクオリティがそのまま広告になる。

`[判断記録]` ターゲットは55歳前後の建築業経営者。SNS（X, Instagram等）は使わない層。集客チャネルはSEO（Google検索）一本に絞る。SNS発信は認知チャネルに含めない。

### 2.2 興味（MOFU）— Before/After体験

`[汎用]` **申込前にAfterを体験させる。** これが「めんどくさい」を超える唯一の方法。

```
LP上のBefore/After表示:

1. 「写真を送る前」の状態を見せる（名刺だけ、サイトなし）
2. テンプレートデモページで完成サイトを見せる
3. 「これがうちの会社のサイトになるのか」

→ 相談不要。見るだけで完成イメージが湧く
→ デモページが営業マン代わり
→ 「やってみたい」のハードルがゼロ
```

`[判断記録]` 画像アップロード型プレビューは撤廃。ターゲット層（55歳・建築業経営者）には操作ハードルが高い。完成デモを見せるだけで十分に感情が動く。

### 2.3 申込（BOFU）— /start ページ

`[汎用]` 申込の目的は「感動させること」。相談なしで即申込できる設計。入力より選択。迷わせない。

```
STEP 0: 業種選択（ワンクリック）
STEP 1: テンプレート選択（iframeプレビュー付き3択）
STEP 2: 会社名入力 → リアルタイムデモサイト表示
STEP 3: ドメイン + メアド → 申込完了

ユーザーの心理:
  STEP 2で「自分の会社名が入ったサイト」が即表示 → 感動
  「これが月¥3,000で持てるの？」→ 信頼
  相談不要・フォーム入力だけ → ハードル最低
```

`[判断記録]` 「まず相談」は建築業経営者にとってハードルが高い。電話・メール相談を挟まず、フォーム完結で即申込できる設計にする。STEP 2のリアルタイムデモが最大のコンバージョンドライバー。

### 2.4 アップセル（ライト→ミドル→プレミアム）

`[汎用]` **不満ではなく「もっと良くしたい」で引き上げる。**

```
アップセル経路:
  会員ダッシュボード（/member/[orderId]/settings）から
  プラン比較を常時表示

  ライト（¥3,000/月）→ ミドル（¥8,000/月）:
    - 月次更新が1回→3回に
    - 追加機能の解放

  ミドル（¥8,000/月）→ プレミアム（¥15,000~/月）:
    - 月次更新が無制限に
    - フルカスタム対応

絶対にやらないこと:
  - ライトの機能を意図的に制限する
  - 「ライトでは物足りないでしょ？」的な煽り
  - しつこい営業メール
```

### 2.5 継続（Retention）

`[汎用]` 解約を防ぐのは「使い続ける理由」を提供すること。月次更新が自然な接点になる。

```
継続の仕組み:
  - ライト: 月1回の更新依頼（テキスト・画像差し替え）
  - ミドル: 月3回の更新依頼
  - プレミアム: 無制限の更新依頼
  - 編集依頼フォーム（/member/[orderId]/edit-request）で簡単に依頼
  - 更新のたびにサイトが良くなる → 解約理由がなくなる
```

## 4. 「最大の競合 = 無行動」への対策

`[汎用]`

| 無行動の理由 | 対策 |
|---|---|
| めんどくさい | /startのSTEP 2でリアルタイムデモ。「選ぶだけでサイトができる」を証明 |
| お金かけたくない | 制作費0円。月¥3,000から。初期費用なしの安心感 |
| ホームページなくても困ってない | Before/After事例で「サイトがあるとこう変わる」を見せる |
| パソコンが苦手 | 入力より選択。ワンクリックで業種・テンプレを選ぶだけ |
| 何から始めればいいか分からない | STEP 0〜3の順番通りに進むだけ。迷う余地がない |
