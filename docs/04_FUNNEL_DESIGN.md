## 2. 各段階の設計

### 2.1 認知（TOFU）

`[固有]`

| チャネル | コンテンツ | 目的 |
|---|---|---|
| X(@ando_lyo) | AIアート×サイト制作のノウハウ投稿 | 認知+権威性構築 |
| note（無料記事） | テンプレ制作過程、AIアートTips | SEO+ファン化 |
| note（¥1,000記事） | テンプレコード配布（Xプロモで無料） | 拡散+リスト獲得 |
| テンプレデモ | 10種のデモページ | 「こんなサイトが持てる」 |

`[汎用]` 認知段階では「売る」のではなく「見せる」。制作物のクオリティがそのまま広告になる。

### 2.2 興味（MOFU）— プレビュー体験

`[汎用]` **購入前にAfterを体験させる。** これが「めんどくさい」を超える唯一の方法。

```
LP上のプレビュー機能:

1. ユーザーが画像を1枚アップロード（or URL入力）
2. テンプレートを選択
3. 3秒でテンプレートに画像がはめ込まれた状態が表示される
4. 「これが¥980で持てます」

→ 決済前・個人情報不要
→ クライアントサイド処理（Canvas API / CSS合成）
→ 「やってみたい」のハードルがゼロ
```

`[判断記録]` 論理（「サイトがあると仕事が来る」）では「めんどくさい」は超えられない。感情（「自分の作品がこう見える」）で超える。

### 2.3 体験（¥980）

`[汎用]` 体験の目的は「感動させること」。100点のサイトを出す。70点にして不満を残す設計は**やらない**。

```
ユーザーの心理:
  「5分で自分のサイトができた」→ 感動
  「これ¥980で持てるの？」→ 信頼
  「でも、ここちょっと変えたいな…」→ 自然な欲求
```

`[判断記録]` 初期設計では「70点にして不満を残す」案があったが、却下。ユーザーをわざと困らせる設計はバレる。正しいのは「100点だけど、120点を見せる」。

### 2.4 アップグレード（¥980→¥2,980）

`[汎用]` **不満ではなく「憧れ」で引き上げる。**

```
手段:
  1. 自動メール配信（6回シリーズ）
  2. おまかせプランのBefore/After事例を見せる
  3. 「同じテンプレなのにこんなに違う」を体感させる
  4. 独自ドメインの価値を伝える
  5. 1ヶ月以内なら¥980引き（緊急性）

絶対にやらないこと:
  - ¥980の機能を意図的に制限する
  - 「¥980では物足りないでしょ？」的な煽り
  - しつこいメール（6回で終わり）
```

### 2.5 高単価（プレミアム）

`[汎用]` プレミアムは「来た人だけに対応」。LP上では「要相談」とだけ表示。積極的に売り込まない。¥2,980の顧客から自然に「もっとやってほしい」と言われたときに提案。

### 2.6 Phase 2移行（コンサル）

`[汎用]` Phase 1の全資産がPhase 2の商品になる構造を意識して、Phase 1の全行動を設計する。

## 4. 「最大の競合 = 無行動」への対策

`[汎用]`

| 無行動の理由 | 対策 |
|---|---|
| めんどくさい | プレビュー体験で「5分でできる」を証明 |
| お金かけたくない | ¥980買い切り。月額じゃない安心感 |
| SNSで十分 | 「流れていくだけでいいの？」で問題提起 |
| 必要性を感じない | Before/After事例で「サイトがあるとこう変わる」を見せる |
| 何から始めればいいか分からない | フォーム入力だけ。選択肢を最小に |
