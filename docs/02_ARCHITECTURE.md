## 3. データフロー

### 3.1 ¥980 サイト自動生成フロー

```
[ユーザー]
  │
  ├─ 1. LP上のフォームに入力
  │     ・名前、ジャンル、テンプレ選択、画像、自己紹介、SNSリンク
  │
  ├─ 2. Stripe Checkoutで¥980決済
  │
  ▼
[Stripe]
  │
  ├─ 3. 決済完了 → Webhook発火
  │
  ▼
[GAS]
  │
  ├─ 4. Webhookを受信
  ├─ 5. 顧客情報をスプレッドシートに記録
  ├─ 6. フォームデータを整形
  ├─ 7. Claude Code CLIに生成指示を送信
  │     ・テンプレートID
  │     ・ユーザーデータ（名前、画像URL、テキスト等）
  │
  ▼
[Claude Code CLI]
  │
  ├─ 8. テンプレートをクローン
  ├─ 9. ユーザーデータを注入（テキスト、画像パス、色、レイアウト）
  ├─ 10. 画像を最適化（WebP変換、リサイズ）
  ├─ 11. ビルドテスト
  │
  ▼
[GitHub]
  │
  ├─ 12. 新リポジトリ作成 or 既存リポに新ブランチ
  ├─ 13. コードをpush
  │
  ▼
[Vercel]
  │
  ├─ 14. 自動ビルド・デプロイ
  ├─ 15. URL発行（username.vercel.app）
  │
  ▼
[GAS]
  │
  ├─ 16. 完了メールをユーザーに送信
  ├─ 17. 自動メール配信スケジュール登録（6回シリーズ）
  │
  ▼
[完了]
```

### 3.2 ¥2,980 AIカスタマイズフロー

```
[ユーザー]
  │
  ├─ 1. 編集フォームに変更内容を入力
  │
  ▼
[GAS]
  │
  ├─ 2. Claude Code CLIに解釈指示を送信
  │
  ▼
[Claude Code CLI]
  │
  ├─ 3. 変更内容を解釈
  ├─ 4. 「この変更でよろしいですか？」をメールで返信
  │
  ▼
[ユーザー]
  │
  ├─ 5. 承認（メール返信 or フォーム）
  │
  ▼
[GAS]
  │
  ├─ 6. 承認を検知
  ├─ 7. Claude Code CLIに実装指示を送信
  │
  ▼
[Claude Code CLI]
  │
  ├─ 8. サイトを改変
  ├─ 9. GitHub push → Vercel自動デプロイ
  │
  ▼
[GAS]
  │
  ├─ 10. 完了通知メール
```

## 5. メール配信設計

### 5.1 自動配信スケジュール（¥980購入者向け）

| # | タイミング | 件名 | 目的 |
|---|---|---|---|
| 1 | 購入直後 | サイトが完成しました！ | URL通知・初期感動 |
| 2 | 3日後 | サイトをSNSに載せましたか？ | 行動促進 |
| 3 | 1週間後 | おまかせプランの事例紹介 | アップグレード種まき |
| 4 | 2週間後 | サイトをもっと良くする3つのヒント | 欲求刺激 |
| 5 | 3週間後 | アップグレード割引は残り1週間 | 緊急性 |
| 6 | 1ヶ月後 | 割引期間が終了しました | 最終通知 |

### 5.2 実装

- GAS の時限トリガー（ScriptApp.newTrigger）
- テンプレートメールをスプレッドシートで管理
- 送信ログもスプレッドシートに記録

## 6. セキュリティ設計

### 6.1 データの分類

| データ | 保存場所 | アクセス権 |
|---|---|---|
| メールアドレス | GAS Spreadsheet | Lyoのみ |
| 作品画像 | Vercel（各サイトのpublic/） | 公開 |
| 決済情報 | Stripe（保持しない） | Stripe管理 |
| サイトソースコード | GitHub（private repo） | Lyoのみ |

### 6.2 認証

- 会員コンテンツ: パスワード保護 or 限定URL（MVP段階）
- 将来: Vercel Edge Middleware + 認証
