## 2. 解約条件

`[汎用]` 「消さない」方式。競合との差別化ポイント。

| 項目 | 内容 |
|---|---|
| 解約タイミング | いつでも（縛りなし） |
| サイト | 公開されたまま残る |
| カスタマイズ | 停止 |
| 会員コンテンツ | アクセス停止 |
| 独自ドメイン | 維持（顧客のドメイン） |
| 再開 | いつでも可能 |

`[判断記録]` 「消す」方式（ビズサイ等）はロックインが強いが、クリエイターに嫌われる。「残す・止まる」方式は信頼獲得+再開動機の両立。サイトが古くなっていく → 本人が一番気になる → 再開。

## 3. 収益シミュレーション

### 3.1 コスト構造

```
固定費:
  Vercel Pro:    ¥3,000/月
  Claude API:    ¥30,000/月（変動）
  合計:          ¥33,000/月

変動費:
  Stripe手数料:  3.6%/取引
```

### 3.2 損益分岐

```
¥980のみ:    ¥33,000 ÷ ¥980 = 34人（買い切りなので初月のみ）
¥2,980のみ:  ¥33,000 ÷ ¥2,980 = 12人/月
混合想定:     ¥980×20人 + ¥2,980×5人 = ¥34,500 → ほぼ分岐
```
