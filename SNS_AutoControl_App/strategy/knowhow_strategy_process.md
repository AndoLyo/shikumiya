## 1. 問題発見のプロセス

### きっかけ: みかっちコンサル
- 自分では気づけなかった「届ける先が間違ってる」という指摘
- 売れた実績（SDプロンプト集）があるのに、同じ池で釣り糸を垂らしていた

### ラーメン屋理論（みかっち）
```
× 「ラーメンを作る過程を見せる人」→ 何しに来るの？
○ 「美味いラーメンを作る人」→ 食べに来る
```
**ノウハウ:** 自分の定義が「やってること説明」になっていたら危険信号。
お客さんにとっての価値で自分を定義し直す。

### 自分視点 vs 相手視点の気づき
- 自分: 「仕組みを作れます」「画像生成歴3年」
- 相手: 「作品はあるのに発信で止まってませんか？」

**ノウハウ:** コンテンツを作ったら「これを読んだ相手は"自分のこと"だと感じるか？」をチェック。
自分視点で書く癖がある場合、必ず相手視点に翻訳するステップを入れる。

## 2. ピボット方針の決め方

### ステップ1: 自分の武器を棚卸しする
```
AI画像生成歴3年 / 全ツール対応 / プロンプト設計 / Claude Code開発力
SDプロンプト集（実績あり: ワンクリ入力、作業時間1/10）
SNS自動化システム / note記事の仕組み / Discord運営経験
```

### ステップ2: 売れた実績から逆算する
- SDプロンプト集が売れた → 解決した課題は「プロンプト考えるのめんどい・時間かかる」
- 買った人はAI界隈の人 → でも同じ課題はクリエイター界隈にもある

### ステップ3: 釣り場を変える
- AI界隈: 競合だらけ、同じトーンの投稿が毎日流れる
- クリエイター界隈: 同じ商品を「相手の言葉」で届ければ競合が少ない

**ノウハウ:** 商品を変えるのではなく、届ける場所と言葉を変える。
「Claude Code」→「ポートフォリオ 作り方」のように翻訳する。

## 3. アカウント戦略の設計プロセス

### 判断基準を先に決める
```
この投稿を見て「欲しい」と思う人は？
├── AIツールを使いこなしたい人 → 既存アカ
└── 作品はあるけど発信で困ってる人 → 新アカ
```

**ノウハウ:** 2アカウント運用する時は「迷った時の判断基準」を1文で決めておく。
記事/投稿を作るたびに毎回考えなくて済む。

## 4. プロフィール設計のプロセス

### 構造（相手視点で組む）
```
1行目〜2行目: 相手の悩み（「自分のことだ」と思わせる）
3行目: それを解決する（自分の提供価値）
4行目: 信頼の根拠（実績・経験）
5行目: CTA（押し売りではなく自然な誘導）
```

### 表示名の考え方
- × 自分視点:「画像生成×仕組みづくり」（やってること説明）
- ○ 相手視点:「作る以外、全部仕組みで。」（相手が得られる価値）

**ノウハウ:** 表示名は「この人をフォローしたら自分に何が起きるか」が伝わるものにする。

## 5. ファネル設計のプロセス

### 低単価を排除した理由
- 情報がすぐ古くなる → 維持コストが高い
- 「今だけ」のリーチで集客ハードルを下げたほうが効率的
- まず信頼関係 → 有料級を無料で出して証明する

### 段階構造の核心
```
Phase 0: 認知（知ってもらう）
Phase 1: 信頼（有料級を無料で → 引き換えに集客協力）
Phase 2: 集客（期間限定・リポスト企画で爆発）
Phase 3: 高単価（単体 or パッケージ → 自分の時間を売らない）
Phase 4: ストック（メンバーシップで積み上げ）
```

**ノウハウ:** 無料→高単価に直接飛ぶ。間を繋ぐのは「信頼」。
低単価の階段を作るより、無料で信頼を積んで一気に高単価に持っていくほうが効率的。

### 価格設計の心理
- 単体: 必要なものだけ選べる → ただし割高
- パッケージ: 全部入り + 特典 → 圧倒的お得感
- 「単体3つ買うより1万円安い」→ パッケージに誘導される設計

## 6. プロモーション設計のプロセス

### オーガニック×クロスプラットフォーム（インフルエンサー不要）
- バズ狙い投稿4パターン: 共感・Before/After・数字・スレッド
- 拡散企画: リポスト配布・感想プレゼント・引用RT誘導
- クロス: IG/Threads→X、note→Xスレッド→note誘導
- X広告: 反応良い投稿だけをブースト（少額テスト）

### SNS_AutoControl_Appの活かし方
- 「量」は自動化でカバー → 日常投稿を自動生成＆投稿
- 「質」は手動で磨く → バズ狙い・企画投稿は手動設計
- 2アカウント × 3プラットフォーム = 同時運用コストほぼゼロ

**ノウハウ:** 自動化システムを持っていることが最大の武器。
プロモーションの「やるべきこと」を設計すれば、実行は仕組みに任せられる。

## 7. みかっちの3つの問い（汎用フレームワーク）

何かを始める前に必ず答える:
1. **誰に向けて？** → ターゲットを具体的に
2. **なんのために？** → 相手にとっての価値で
3. **なぜわたしが？** → 実績・経験・独自性で

**ノウハウ:** この3つに答えられないなら、まだ始める段階じゃない。
特に「なぜわたしが？」に答えられないと、競合に埋もれる。

## 8. 手順書レベル設計（みかっち方針）

レベルは過程で決めるのではなく、最初に決めるもの:
- 初心者向け → 小学生でもわかる
- 中級者向け → 中学生でもわかる
- 上級者向け → 高校生でもわかる

**ノウハウ:** 「わかりやすい」の定義は相手のレベルによって変わる。
先にレベルを決めてから書き始めること。
