Section 01

問題発見 — 「届け先が間違っている」と気づく

事業が伸び悩んだとき、多くの人は「商品が悪い」と思い込みます。 でも実際は、商品はそのまま、届ける先と伝え方が間違っているだけかもしれません。

僕の場合、過去にプロンプト集が売れた実績がありました。でも同じ界隈で同じように発信し続けていた。 同じ池にずっと釣り糸を垂らしていたんです。

「ラーメン屋理論」で考える

「ラーメンを作る過程を見せる人」
→ お客さんは何しに来るの？
「美味いラーメンを作る人」
→ 食べに来る
Knowhow

自分の定義が「やってること説明」になっていたら危険信号。
お客さんにとっての価値で自分を定義し直す。

自分視点 vs 相手視点

自分視点:「仕組みを作れます」「画像生成歴3年」
相手視点:「作品はあるのに発信で止まってませんか？」
Knowhow

コンテンツを作ったら「これを読んだ相手は"自分のこと"だと感じるか？」をチェック。
自分視点で書く癖がある場合、必ず相手視点に翻訳するステップを入れる。

Section 02

ピボット方針 — 3ステップで届け先を変える

  1. 自分の武器を棚卸しする
    テンプレート — 武器の棚卸し あなたのスキル・経験を全て書き出す: ・専門スキル（例: AI画像生成歴3年 / プロンプト設計） ・開発力（例: Claude Code開発 / 自動化システム構築） ・実績（例: プロンプト集販売 / 作業時間を1/10に短縮） ・運営経験（例: コミュニティ運営 / note記事執筆）
  2. 売れた実績から逆算する

    何が売れたか → それが解決した課題は何か → 同じ課題を持つ別の層はいないか？

    僕の場合: プロンプト集が売れた → 解決した課題は「プロンプトを考えるのが面倒・時間がかかる」 → この課題はAI界隈だけでなくクリエイター界隈にもある

  3. 釣り場を変える
    • 既存の界隈: 競合だらけ、同じような投稿が毎日流れる
    • 別の界隈: 同じ商品を「相手の言葉」で届ければ競合が激減する
Knowhow

商品を変えるのではなく、届ける場所と言葉を変える。
「Claude Code」→「ポートフォリオ 作り方」のように翻訳するだけで刺さる相手が変わる。

Section 03

アカウント戦略 — 迷わない判断基準を1つ持つ

複数アカウントを運用するとき、毎回「この投稿はどっちで出す？」と迷っていたら時間の無駄です。
判断基準を1文で決めておけば、一瞬で分類できます。

テンプレート — アカウント判断基準 この投稿を見て「欲しい」と思う人は？ ├── ツールを使いこなしたい人 → アカウントA └── 作品はあるけど発信で困っている人 → アカウントB
Knowhow

2アカウント運用する時は「迷った時の判断基準」を1文で決めておく。
記事や投稿を作るたびに毎回考えなくて済む。

Section 04

プロフィール設計 — 相手視点で組む5行構造

テンプレート — プロフィール構造 1行目〜2行目: 相手の悩み（「自分のことだ」と思わせる） 3行目: それを解決する（あなたの提供価値） 4行目: 信頼の根拠（実績・経験・数字） 5行目: CTA（押し売りではなく自然な誘導）

表示名の考え方

「画像生成×仕組みづくり」
→ やってること説明（自分視点）
「作る以外、全部仕組みで。」
→ 相手が得られる価値（相手視点）
Knowhow

表示名は「この人をフォローしたら自分に何が起きるか」が伝わるものにする。

Section 05

ファネル設計 — 無料から高単価に直接飛ぶ

低単価商品を排除した理由

  • 情報はすぐ古くなる → 維持コストが高い
  • 「今だけ無料」のリーチで集客ハードルを下げたほうが効率的
  • まず信頼関係を構築 → 有料級を無料で出して自分の実力を証明する

段階構造

Phase 0認知 — まず知ってもらう
Phase 1信頼 — 有料級を無料で出す。引き換えに拡散協力をもらう
Phase 2集客 — 期間限定・リポスト企画で一気にリーチを広げる
Phase 3高単価 — 単体 or パッケージ販売。自分の時間を売らない
Phase 4ストック — メンバーシップで月額の積み上げ収益をつくる
Knowhow

無料→高単価に直接飛ぶ。間を繋ぐのは「信頼」。
低単価の階段を作るより、無料で信頼を積んで一気に高単価に持っていくほうが効率的。

価格設計の心理テクニック

  • 単体: 必要なものだけ選べる → ただし割高に設定
  • パッケージ: 全部入り＋特典 → 圧倒的なお得感
  • 「単体3つ買うより1万円安い」→ 自然にパッケージへ誘導される設計
Section 06

プロモーション設計 — インフルエンサー不要の広げ方

オーガニック × クロスプラットフォーム戦略

  • バズ狙い投稿4パターン: 共感・Before/After・数字・スレッド
  • 拡散企画: リポスト配布・感想プレゼント・引用RT誘導
  • クロス投稿: Instagram/Threads → X、note → Xスレッド → note 誘導
  • 少額広告: 反応が良い投稿だけをブーストしてテスト

SNS自動化システムの活かし方

  • 「量」は自動化: 日常投稿を自動生成＆投稿でカバー
  • 「質」は手動: バズ狙い・企画投稿は手動で丁寧に設計
  • 2アカウント × 3プラットフォーム = 同時運用コストほぼゼロ
Knowhow

自動化の仕組みを持っていることが最大の武器。
プロモーションの「やるべきこと」を設計すれば、実行は仕組みに任せられる。

Section 07

何かを始める前の「3つの問い」

新しいプロジェクト・商品・コンテンツを始める前に、必ずこの3つに答えてください。
答えられないなら、まだ始める段階ではありません。

フレームワーク — 着手前の3つの問い 1. 誰に向けて？ → ターゲットを具体的に 2. なんのために？ → 相手にとっての価値で答える 3. なぜわたしが？ → 実績・経験・独自性で答える
Knowhow

特に「なぜわたしが？」に答えられないと、競合に埋もれる。
ここが弱いなら、まず実績をつくるフェーズに戻る。

Section 08

手順書レベル設計 — 「わかりやすさ」の基準を先に決める

コンテンツを作る前に「誰に向けて書くか」でレベルを決めます。
書き始めてから調整するのではなく、最初にレベルを固定するのがコツです。

テンプレート — レベル設定 初心者向け → 小学生でもわかるレベル 中級者向け → 中学生でもわかるレベル 上級者向け → 高校生でもわかるレベル
Knowhow

「わかりやすい」の定義は相手のレベルによって変わる。
先にレベルを決めてから書き始めること。
